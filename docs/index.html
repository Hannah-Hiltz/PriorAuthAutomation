How the pipeline works

Four stages convert a free-text clinical note into a structured, auditable decision. Each stage is swappable and individually evaluable.

Documentation quality vs. decision

Why this matters

Physicians spend ~16 hours/week on PA paperwork and 94% report delays lead to care abandonment. This is a workflow problem with direct clinical consequences.

Where the savings come from

Three compounding levers: cheaper unit cost per PA, fewer appeals/overturns (better letters cite criteria correctly), and faster cycle time (fewer abandoned authorizations).

Operational safety

Not every case is auto-decided. Safety-flagged PENDING routing (e.g. suicidality, AUD+bupropion) preserves clinician oversight — a required feature for payer adoption.

Evaluation & what I'd build next

Grounded, measurable, and honest about the ceilings.

How I measured quality

BERTScore F1 against gold-standard justification letters: 0.687 full letter, 0.757 rationale-only. Classification accuracy evaluated against 25 labeled cases. Documentation-gap detection evaluated as a retrieval task.

Next iteration

Expand to 500 cases with payer-specific policy corpora, add peer-to-peer dialog simulation, benchmark against production vendor tools on turnaround and overturn rate, and ship a feedback loop so reviewer overrides retrain the rule weights.
